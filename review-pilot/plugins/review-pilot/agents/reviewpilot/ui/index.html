Analysis
✓ Code pattern pre-scan
i Code Pattern Pre-scan

Before the AI runs, a fast script scans every changed line for common issues — at zero AI cost:

•Deprecated Angular ::ng-deep / /deep/
•Hardcoded SCSS colours when $variables exist
•Legacy var, loose ==, console.log
•jQuery usage in modern Angular stacks
•Missing doc comments on new exports
•SELECT *, NOLOCK, printStackTrace

The AI then confirms or dismisses each candidate — saving credits by not discovering what a script already caught.

✓ Recommended: leave ON
✓ Include code diffs
i Include Code Diffs

Attaches the actual git diff for each reviewed file (trimmed to ≈ 6 KB per file) directly inside the AI prompt.

Without this, the AI only sees file names and change counts. With it, the AI can read the exact lines added or removed — producing much more precise findings and fix proposals.

Turn OFF only if your branch has very large diffs and you are running into token-limit errors.

✓ Recommended: leave ON
Processing
Enable chunking
i Enable Chunking

When enabled, large branch diffs are automatically split into sequential AI calls that share one review session, keeping each call within the token budget.

When disabled (default), the entire diff is sent in a single AI call — best for most branches and produces a cleaner single-pass report.

Turn ON only if you hit token-limit errors on a very large branch.

Default: OFF

Pre-scan runs in milliseconds at no AI cost · diffs give the AI full context · together they make each review faster and cheaper.
